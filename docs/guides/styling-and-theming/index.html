Styling and Theming

Use the bundled stylesheet when you want a fast default look. If you need the tour to match your product more closely, Siteguide.js gives you a few simple customization layers.

Re-skin the generated markup with classPrefix​

classPrefix changes the generated class namespace from siteguide-* to your own prefix.

const tour = new Tour({
    classPrefix: 'acme-guide',
});

This is the cleanest way to swap in a complete custom stylesheet.

Add step-specific classes​

Use PopupCustomization when one step should look different from the rest:

popup: {
    type: 'text',
    title: 'Branded step',
    text: 'Attach classes only for this popup.',
    customization: {
        class: 'brand-popup',
        headerClass: 'brand-popup__header',
        contentClass: 'brand-popup__content',
        footerClass: 'brand-popup__footer',
    },
}

Replace the close icon​

closeIcon accepts an HTMLElement, so you can provide your own icon or button content:

const closeIcon = document.createElement('span');
closeIcon.textContent = '×';

const tour = new Tour({
    closeIcon,
});

Translate content tokens​

translateFn is useful when your step content stores translation keys instead of final text.

const tour = new Tour({
    translateFn: (token) => dictionary[token] ?? token,
});

Recommended styling workflow​

  1. Start with the bundled stylesheet.
  2. Add per-step customization classes where only a few steps differ.
  3. Switch to classPrefix when the entire popup should follow your own design system.
