# Navigation and Controls

Use these options when you want the tour to feel more natural in your product.

## Closing rules

`TourConfig.close` controls how users can exit the tour:

```js
const tour = new Tour({
    close: {
        button: true,
        clickout: false,
        esc: true,
    },
});
```

- `button` shows or hides the close button in the popup header.
- `clickout` closes when the user clicks outside the popup.
- `esc` closes on the `Escape` key.

## Keyboard navigation

Enable left and right arrow navigation with:

```js
const tour = new Tour({
    keyboardControl: true,
});
```

This adds:

- `ArrowRight` -> `next()`
- `ArrowLeft` -> `prev()`

## Scrolling to the active target

`scrollTo` can be:

- `false` to disable scrolling;
- `true` to use the browser default;
- a `ScrollIntoViewOptions` object for explicit behavior.

```js
scrollTo: {
    behavior: 'smooth',
    block: 'center',
    inline: 'nearest',
}
```

## Arrow, highlight, interaction, and background

These sections control the visual treatment around the active step:

```js
const tour = new Tour({
    arrow: {
        disable: false,
        padding: 24,
    },
    highlight: {
        disable: false,
        padding: 8,
    },
    interaction: {
        disable: false,
    },
    background: {
        disable: true,
    },
});
```

What each option controls:

- `arrow` controls the popup arrow and its spacing.
- `highlight` draws attention to the active target.
- `interaction` controls whether the highlighted area stays interactive.
- `background` controls the dimmed backdrop behind the popup.

## Animation

Use `animation.class` and `animation.delay` when you want the popup motion to match your product styling:

```js
animation: {
    class: 'siteguide-animation',
    delay: 400,
}
```
