# Quick Start

Use this example when you want a quick, realistic starting point. It creates a two-step tour and starts it from a button click.

```js title="app.js"
import 'siteguide.js/dist/css/siteguide.css';
import { Tour } from 'siteguide.js';

const tour = new Tour({
    scrollTo: {
        behavior: 'smooth',
        block: 'center',
        inline: 'center',
    },
    progress: {
        disable: false,
    },
});

tour.addSteps([
    {
        host: '.hero-cta',
        popup: {
            type: 'text',
            title: 'Start here',
            text: 'Introduce the primary action before users explore the rest of the page.',
        },
    },
    {
        host: '.settings-link',
        popup: {
            type: 'text',
            title: 'Configuration',
            text: 'Guide users to the place where they can tune the experience later.',
            position: 'left',
        },
    },
]);

document.querySelector('#start-tour')?.addEventListener('click', () => {
    tour.start();
});
```

## What this example does

1. `new Tour()` creates a tour instance with your chosen options.
2. `addSteps()` adds two steps in the order you want to show them.
3. `start()` opens the first step when the user clicks the button.
4. `next()`, `prev()`, and `goTo()` are available when you want custom navigation later.

## Common next steps

- Add `buttonList` when you need custom step actions.
- Use `type: 'custom'` when the popup body should be a DOM node you control.
- Set `keyboardControl: true` to support left and right arrow navigation.
- Pass a `translateFn` if your app already has localization tokens.
