# Additional Exports

Most projects only need `Tour` and the main configuration types. Siteguide.js also exports a few secondary utilities for advanced use cases.

## `TourStep`

`TourStep` represents the current step object that you can inspect through `tour.activeStep`.

Typical use:

- read information about the active step in an event callback;
- check the current step index or popup data while debugging;
- build lightweight integrations around the running tour.

For most applications, you do not need to create `TourStep` yourself.

## `Siteguide`

`Siteguide` is a global status object that tells you whether a tour is running.

```js
import { Siteguide } from 'siteguide.js';

if (Siteguide.isActive) {
    console.log(Siteguide.activeTour);
}
```

Use it when you need app-wide awareness of the active tour.

## `createElement`

`createElement` is a small convenience helper for creating DOM elements with class names.

```js
import { createElement } from 'siteguide.js';

const badge = createElement('span', ['tour-badge']);
badge.textContent = 'New';
```

It is optional. Standard DOM APIs work just as well.
